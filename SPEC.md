# SPEC — CSVエクスポート機能の拡張

## 目的

現行の「CSVエクスポート」（メニュー押下で即DL、フィルタ済み対戦を19列フル出力）を、
設定モーダル経由で「出力種別・列選択・範囲」を選べる形に拡張する。

## 受入条件

1. メニュー「CSVエクスポート」押下で **設定モーダル** が開く（即DLしない）。
2. モーダルで **出力種別** をラジオで1つ選べる：
   - `対戦明細`（現行19列ベース、列ON/OFF選択付き）
   - `ポケ別勝率`
   - `コンボ別勝率`
   - `相性マトリクス`
   - `期間サマリ`
3. `対戦明細` 選択時のみ **列ON/OFFチェックボックス**（19列）が表示され、選択列のみ出力。デフォルト全ON。
4. 列ON/OFF設定は **localStorage 永続化**（次回モーダルでも保持）。キー: `pokemon-export-columns`。
5. **範囲指定**: モーダル内で以下を AND 結合で絞り込み。
   - ルール: `全ルール` + battles に存在するルール
   - シーズン: `全シーズン` + battles に存在するシーズン
   - **日付範囲**: 開始日・終了日（`input[type=date]` 2つ）。`b.date` が範囲内のものを抽出。
     - 片方のみ指定可（開始日のみ=以降、終了日のみ=以前）。両端含む（inclusive）。
   - すべて未指定（全ルール/全シーズン/日付空）なら全件相当。
6. 集計CSV（コンボ別/相性）は **5戦未満の閾値なしで全件出力**（画面表示の MIN_BATTLES=5 は適用しない）。
7. 「ダウンロード」ボタンで選択種別の単一CSVをDL。BOM付きUTF-8、ファイル名 `pokemon-battle-log-<種別>-<日付>.csv`。
8. 対象0件のときはトーストで通知しDLしない。

## 集計CSVの列設計

- **対戦明細**: 現行19列（日付/ルール/シーズン/結果/レート/レート差/自分のパーティ/自分の持ち物/選出/相手のパーティ/相手の持ち物/相手選出/お気に入り/タグ/選出意図/勝因・敗因/立ち回り・分岐点/改善点・TODO/旧メモ）から選択列のみ。
- **ポケ別勝率（自分選出基準）**: `ポケモン, 選出回数, 勝ち, 負け, 勝率%`
- **コンボ別勝率**: `種別(自分ペア/自分トリオ/相手ペア/相手トリオ), 組み合わせ, 試行数, 勝ち, 負け, 勝率%`（1ファイルに4種別を縦結合）
- **相性マトリクス**: ロング形式 `自分ポケモン, 相手ポケモン, 試行数, 勝ち, 負け, 勝率%`（相手軸はパーティ基準）
- **期間サマリ**: `ルール, シーズン, 総戦数, 勝ち, 負け, 引分, 勝率%, 最終レート`

※ 勝率は「勝ち/(勝ち+負け)」、引分・未確定は分母から除外（既存ロジックに合わせる）。

## 実装方針

- **集計ロジックの分離**: `stats.js` の集計ロジックはDOM描画と密結合。CSV用に、対戦配列を受け取り集計データ（配列）を返す純粋関数群を新設する。
  - 案: 新規 `export-data.js` に `aggregatePokeStats(battles)`, `aggregateCombos(battles)`, `aggregateMatchup(battles)`, `aggregatePeriodSummary(battles)` を実装（stats.js のロジックを移植・流用、閾値なし）。
- **modal.js**: 現行 `exportCSV()` を `openExportModal()`（モーダル表示）に置換。実DLは `runExport()`。CSV組み立て（BOM・クォート・改行）は現行ヘルパを共通化。
- **index.html**: エクスポート設定モーダルのHTML追加（種別ラジオ / 列チェックボックス / ルール・シーズンセレクト / ダウンロード・キャンセルボタン）。
- **events.js**: モーダル内要素のリスナー登録（種別変更で列セクション表示切替、ダウンロード、キャンセル）。
- ルール/シーズンのドロップダウンは battles からユニーク値を収集して生成。

## 非目標

- TSV・区切り文字変更、xlsx 出力
- 新規データ列の追加（既存19列のON/OFFのみ）
- グラフ画像のエクスポート
- 複数種別の一括DL（種別は1つ選択）
- 画面フィルタ（結果/期間/タグ）との連動（範囲はモーダル内ルール/シーズンのみ）

## 例外・境界

- battles 0件 / 選択範囲0件 → トースト通知、DLなし。
- 日付範囲: 開始日 > 終了日（逆転）→ 0件扱い（トースト通知）。`b.date` 空の対戦は日付範囲指定時は除外。
- 列が全OFF → ダウンロード無効化 or トースト警告（全OFFはエラー扱い）。
- レート未入力の対戦 → レート/レート差は空欄（現行踏襲）。
- 集計でデータ不足（コンボ2体未満など）→ その行は出力されないだけ（エラーにしない）。

## テスト方針

- 手動: ローカルでブラウザ起動し、各種別のDL内容を目視確認（列ON/OFF反映、範囲絞り込み、閾値なし、空件トースト）。
- localStorage 永続化: モーダル再オープンで列選択が保持されること。

## 互換性

- 既存データ構造・localStorage キー（`pokemon-battle-log`）に変更なし。新規キー `pokemon-export-columns` を追加するのみ。
- 現行のCSV列・順序は `対戦明細` 全ON時に維持（後方互換）。

## ロールバック

- 機能は modal.js / index.html / events.js / 新規 export-data.js に閉じる。
  問題時は該当変更を revert すれば現行の即DL挙動に戻せる。

# SPEC — パーティカード D&D 並び替え

## 受入条件

- パーティタブの各カードヘッダーに「☰」ハンドル表示。
- ハンドルからドラッグ開始でカード並び替え。**PC(マウス)のみ**、HTML5 native DnD 使用。
- ドラッグ中: 対象カード半透明、挿入位置を青ライン表示。
- ドロップで `pokemon-party-presets`(localStorage) の配列順を更新→ Firebase 同期(既存 savePresetsData 経由)。
- dragend/Escape(ブラウザ標準) で中止、順序変更なし。
- ハンドル以外(編集/削除/記録ボタン、カード本体)のクリックは従来動作維持。

## 非目標

- バトル一覧の並び替え。
- 複数選択 D&D。

## 例外

- presets が 0/1 件 → ドラッグ可能だが順序変化なし。
- 同一位置ドロップ → 何もしない(再描画もスキップ)。

## 互換性

- データ構造変更なし。並び順のみ presets 配列順に依存(既存と同じ)。
