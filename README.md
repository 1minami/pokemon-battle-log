# ポケモン対戦記録

ポケモン対戦の記録を管理するスプレッドシート型Webアプリ。ポケモンチャンピオンズ **レギュレーションM-A**（2026-04-08〜2026-06-17）の参加可能 213 体に対応。

## 使い方

https://1minami.github.io/pokemon-battle-log/ にアクセス（PC・スマホ対応）。

ローカルで使う場合は `index.html` をブラウザで開くだけでも動作します（サーバー不要）。PWA対応のため、ホーム画面に追加してアプリとして利用可能。

## 機能

- **タブ切り替え** — 「対戦記録」と「統計」をタブで画面切り替え
- **対戦記録の追加・編集・削除** — 日付、ルール、レート（必須・勝敗はレート差で自動判定）、パーティ、選出、メモ
- **お気に入り（ブックマーク）** — 対戦記録に★マークを付けて目立たせる
- **ポケモン検索** — カタカナ / ひらがな / 日本語ローマ字（ヘボン式, 例: `fushigibana`）/ 英語名スラッグ（例: `venusaur`）のいずれでも検索可能。選択後はカーソルが自動で検索欄に戻る
- **レギュレーション別ポケモン絞り込み** — ルールを選ぶとポケモンピッカーが参加可能なポケモンだけに絞られる（レギュレーションM-A: 213体）
- **レート記録 & 勝敗自動判定** — 試合後レートを必須で記録し、直前の記録との差分から勝ち/負け/引き分けを自動判定。クイック調整ボタン（±15/±5）とマウスホイールでの微調整（±1）に対応。結果列には W/L/D + レート差（例: `W +15`）を表示
- **勝率統計** — ヘッダーに勝ち・負け・勝率をリアルタイム表示
- **ポケモン使用統計** — 自分の選出ポケモンごとの勝率を可視化
- **相手ポケモン統計** — 相手パーティの遭遇数・選出数・対戦勝率を可視化
- **フィルター** — ルール別・結果別（レート差ベース）で絞り込み
- **記録追加時の参考パネル** — 対戦モーダル右側に、同じ相手パーティとの過去対戦（選出・勝敗・レート差）を分割レイアウトで表示
- **ソート** — 日付で昇順/降順切り替え
- **タグ/メタ分類** — 「サイクル構築」「積み構築」などのプリセットタグで対戦を分類。タグでフィルタリング可能
- **勝率トレンドグラフ** — 統計タブで時系列の累積勝率推移をCanvasグラフで可視化。パーティ・期間フィルター連動
- **CSVエクスポート** — Excelで開けるUTF-8 BOM付きCSV出力
- **JSONエクスポート/インポート** — 対戦記録+パーティの完全バックアップ・復元。上書きまたは既存データへの追加を選択可能。旧形式（配列のみ）のインポートにも対応
- **前回記録の引き継ぎ** — 新規追加時にルールと自分のパーティを前回の対戦記録から自動セット
- **パーティプリセット** — よく使うパーティを名前付きで保存し、記録追加時にワンクリックで呼び出し
- **ローカルストレージ保存** — ブラウザに自動保存（サーバー不要）。容量超過時はトースト通知で警告
- **対面勝率マトリクス** — 統計タブに、自分の選出×相手パーティ/選出の勝率ヒートマップを表示。色分けで一目で有利不利が分かる。相手軸はタブで「相手パーティ / 相手選出」を切替可能。セルクリックで該当試合を真下に一覧展開
- **クラウド同期（Firebase）** — Googleアカウントでログインし、「☁ 同期」ボタン1つで自動判定（ローカル/クラウドの更新時刻を比較し、必要な方向にアップロード or ダウンロード）。サブテキストに最終同期からの相対時刻表示。両方変更時のみ競合モーダル（「先にダウンロード / 強制アップロード / キャンセル」）。PC・スマホ間でデータ共有可能
- **PWA対応** — ホーム画面に追加してアプリとして利用可能。Service Workerによるオフラインキャッシュ対応
- **スマホ対応** — 768px以下でカード型レイアウトに自動切替。タッチ操作に最適化されたUI
- **キーボードショートカット** — `Ctrl+N` で新規追加、`Esc` でモーダル閉じ

## Firebase セットアップ（クラウド同期を使う場合）

1. [Firebase Console](https://console.firebase.google.com/) でプロジェクトを作成
2. Authentication > Sign-in method で「Google」を有効化
3. Firestore Database を作成（テストモードで開始してOK）
4. プロジェクト設定 > マイアプリ > ウェブアプリを追加し、表示される config を `firebase-config.js` に記入（`firebase-config.example.js` をコピーして作成）
5. Authentication > Settings > 承認済みドメインに GitHub Pages の URL（例: `1minami.github.io`）を追加
6. GitHub Pages デプロイ時は GitHub Secrets に `FIREBASE_API_KEY`, `FIREBASE_AUTH_DOMAIN`, `FIREBASE_PROJECT_ID`, `FIREBASE_STORAGE_BUCKET`, `FIREBASE_MESSAGING_SENDER_ID`, `FIREBASE_APP_ID` を設定

## 記録項目

| 項目 | 説明 |
|------|------|
| 日付 | 対戦日 |
| ルール | レギュレーションM-A（ポケモンチャンピオンズ, 2026-04-08〜2026-06-17）。選択中はポケモンピッカーが参加可能な213体に絞り込まれる |
| レート | 試合後のレート（整数・**必須**）。直前の記録との差分で勝敗を自動判定（+で勝ち / -で負け / ±0で引き分け）。±15/±5 のクイック調整ボタンとマウスホイール（±1）で入力可能 |
| 自分のパーティ | 最大8体（1〜2体の差し替え候補を含めて記録可能） |
| 選出 | 実際に選出した3体 |
| 相手のパーティ | 相手の6体 |
| 相手選出 | 相手の選出 |
| お気に入り | ★マークで記録をブックマーク |
| タグ | 構築タイプ分類（プリセット: 対面構築/サイクル構築/積み構築/天候パ/トリルパ/壁構築/受けループ/ギミック） |
| 選出意図 | なぜその3体を選んだか（相手想定・役割分担など） |
| 勝因・敗因 | 勝ち/負けの決定的要因（キープレイや読みの当否） |
| 立ち回り・分岐点 | 試合中の判断ポイント・別ルートの選択肢 |
| 改善点・TODO | 次回へ向けた反省点・構築調整案 |
| 旧メモ | 細分化前の旧「メモ」欄（既存データのみ読取専用で表示） |
